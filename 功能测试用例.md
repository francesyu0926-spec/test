# 招投标数字平台项目审计详情页功能测试用例

## 1. 截图内容分析

截图展示的是“招投标数字平台”的 AI 审计结果详情页，当前项目为“广州生产基地机电净化工程”。页面主要结构如下：

- 顶部区域：平台 Logo、顶部页签“AI审计结果 / 数据展示看板 / 智能审计大屏”、当前用户信息。
- 左侧菜单：包含“我的账户、我的项目、快控项目”等入口，当前“快控项目”高亮。
- 主内容区：展示项目标题、招标备案类别、投标文件检查、投标文件复核、人员资质检查等审计模块。
- 右侧目录：包含“风险预警”和本项目章节锚点，当前章节高亮。
- 数据形态：普通表格、文件链接、查看操作、绿色“已完成”状态、红色“不合格”状态、蓝色选中矩阵单元格、百分比/短横线等复核结果。

## 2. 测试范围

- 项目审计详情页加载、顶部导航、左侧菜单、右侧目录锚点。
- 招标备案类别中文件类型、文件名称、页数、一审数量、二审数量、目录数量、查看操作。
- 投标文件检查表格中的审查项、页数、AI评审状态、AI评审结果、查看操作。
- 投标文件复核矩阵中的单位维度、审查项维度、百分比/空值、单元格选中高亮。
- 风险预警、异常状态、权限控制、接口异常、空数据、兼容性。

## 3. 测试前置条件

| 编号 | 前置条件 |
| --- | --- |
| P-01 | 测试账号已登录招投标数字平台，并拥有“快控项目”查看权限。 |
| P-02 | 系统存在项目“广州生产基地机电净化工程”或同类项目详情数据。 |
| P-03 | 项目包含招标备案文件、投标文件检查、投标文件复核和人员资质检查数据。 |
| P-04 | 系统存在至少 1 条 AI 审核“不合格”或风险预警数据。 |
| P-05 | 文件预览/下载服务、AI 审核结果接口、章节锚点接口可正常访问。 |

## 5. 回归测试重点

| 回归点 | 验证内容 |
| --- | --- |
| 页面导航 | 顶部页签、左侧菜单、右侧目录锚点均可正常切换和定位。 |
| 数据准确性 | 文件信息、AI 状态、AI 结果、矩阵百分比与数据源一致。 |
| 审计状态 | 绿色已完成、红色不合格、风险预警等状态展示准确。 |
| 文件能力 | 文件名称链接和“查看”入口可正确打开并处理权限异常。 |
| 矩阵交互 | 复核矩阵点击选中、行列定位、横向滚动和空值展示正常。 |
